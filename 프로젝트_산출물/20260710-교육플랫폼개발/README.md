# 교육플랫폼개발 - 프로젝트 산출물 폴더

- 복잡도: 간단 (simple)
- 루트 폴더: 20260710-교육플랫폼개발
- 설명: 빠르게 시작하는 소규모/사이드 프로젝트용. 하위 폴더 없이 5개 대분류만 생성합니다.

## 폴더 구조

```
01_요구사항

02_설계

03_개발

04_테스트

05_산출물
```

## 폴더별 설명

### 01_요구사항

요구사항정의서, 요청사항 목록 등 프로젝트 범위를 정의하는 문서를 보관합니다.

### 02_설계

화면설계, 시스템 구조, DB 설계 등 설계 관련 문서를 보관합니다.

### 03_개발

소스코드 백업/스냅샷, 개발 관련 메모를 보관합니다. 형상관리 도구 연동을 권장합니다.

### 04_테스트

테스트 계획, 테스트케이스, 테스트 결과를 보관합니다.

### 05_산출물

최종 산출물, 배포 결과물, 검수 관련 문서를 보관합니다.

## 운영 규칙 (제안)

1. 버전관리: 파일명은 "문서명_v1.0_YYYYMMDD" 형식 권장. 최종본은 "_최종" 표기.
2. 승인 이력: 산출물 검토/승인 이력은 관련 폴더의 체크리스트에서 통합 관리.
3. 소스코드는 형상관리 도구(Git 등) 연동을 우선하고, 폴더는 스냅샷/백업 용도로 사용.
4. 프로젝트 종료 후 최종 산출물 목록과 함께 보관 기간을 명시.

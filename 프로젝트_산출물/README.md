# 프로젝트 산출물 폴더 구조 (IT/SW 개발 프로젝트)

품질관리자(QA/QC) 관점에서 SDLC 단계별 산출물과 품질관리 활동 기록을 체계적으로 관리하기 위한 표준 폴더 구조입니다.

## 전체 구조

```
00_프로젝트관리
├── 01_착수_계획          - 프로젝트 헌장, 착수보고서, 수행계획서
├── 02_일정_이슈관리       - WBS, 일정표, 이슈/리스크 관리대장
├── 03_회의록             - 정기/수시 회의록
└── 04_변경관리           - 변경요청서(CR), 변경이력

01_요구사항분석
├── 01_요구사항정의서      - 기능/비기능 요구사항 정의서
└── 02_요구사항추적표      - RTM(Requirements Traceability Matrix)

02_설계
├── 01_시스템설계서        - 아키텍처, 시스템 구성도
├── 02_화면설계서          - UI/UX 설계, 화면정의서
├── 03_DB설계서           - ERD, 테이블정의서
└── 04_인터페이스설계서    - 연계 인터페이스 정의서

03_개발
├── 01_소스코드           - 소스코드 형상관리 연동 또는 백업
├── 02_코드리뷰결과        - 코드리뷰 체크리스트, 지적사항
└── 03_개발표준_가이드     - 코딩 표준, 개발 가이드라인

04_테스트
├── 01_테스트계획서        - 단위/통합/시스템/UAT 테스트 계획
├── 02_테스트케이스        - 테스트케이스 명세서
├── 03_테스트결과보고서    - 테스트 수행 결과
└── 04_결함관리대장        - 결함(Defect) 등록/추적 대장

05_품질관리
├── 01_품질관리계획서      - QA/QC 계획서, 품질기준
├── 02_검토_점검체크리스트 - 산출물 검토 체크리스트
├── 03_감리_점검결과       - 내/외부 감리, 정기점검 결과
└── 04_부적합관리대장      - 부적합사항 등록 및 시정조치 이력

06_배포_이관
├── 01_배포계획서
├── 02_배포체크리스트
└── 03_이관확인서

07_교육_매뉴얼
├── 01_사용자매뉴얼
├── 02_운영매뉴얼
└── 03_교육자료

08_검수_인수
├── 01_검수요청서
├── 02_검수확인서
└── 03_최종산출물목록          - 최종 납품 산출물 목록 및 체크리스트

09_사후관리
├── 01_프로젝트종료보고서
└── 02_유지보수계획
```

## 운영 규칙 (제안)

1. **버전관리**: 파일명은 `문서명_v1.0_YYYYMMDD` 형식 권장. 최종본은 `_최종` 표기.
2. **승인 이력**: 각 산출물 폴더 내 승인/검토 이력은 `05_품질관리/02_검토_점검체크리스트`에서 통합 관리.
3. **접근 권한**: 03_개발의 소스코드는 형상관리 도구(Git 등) 연동을 우선하고, 이 폴더는 릴리스 시점 스냅샷 용도로 사용.
4. **폐기/보관**: 프로젝트 종료 후 09_사후관리에 최종 산출물 목록과 함께 보관 기간을 명시.

각 하위 폴더에는 용도를 설명하는 README.md가 포함되어 있습니다.
